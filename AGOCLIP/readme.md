Release Notes: WiMANS-CLIP 细粒度跨模态对齐与槽位解耦重构
1. 编码器底层架构升级 (细粒度序列解锁)
WiFi 编码器 (THAT) 序列化改造：彻底移除了时间维度的全局求和池化 (torch.sum)。为 1D 卷积补充 padding="same" 以解决不同卷积核带来的序列长度不对齐问题；引入“全局频率广播机制”解决时域与频域的物理维度碰撞。最终成功输出保留 150 个时间步的细粒度特征序列 [Batch, 150, 512]。

Video 编码器 (S3D) 时空解绑：绕过原生分类头，仅对视频特征的空间维度（高、宽）进行平均池化 mean(dim=[3, 4])，坚决保留时间帧维度，输出匹配的 [Batch, Time, 512] 视频特征流。

2. 跨模态预训练目标函数重构 (SOTA级软对齐机制)
Token-wise 帧级相似度矩阵：废弃粗暴的二维全局矩阵乘法，使用 einsum 构建高维交叉点积矩阵，实现 WiFi 帧与视频帧的局部对齐。

梯度游离 (Gradient Detach) 与软对齐：使用 tau=0.5 的 Softmax 替代硬性 max 操作，让所有时间帧均可回传梯度。通过 sim.detach() 切断注意力计算与 Loss 计算的梯度耦合，使注意力成为纯粹的对齐向导。

帧级动态加权 (Dynamic Frame Weighting)：引入基于注意力的动态加权求和，彻底消除了背景静默帧对真实动作特征的稀释灾难。

防过拟合与防作弊机制：对温度系数实施硬性钳制 clamp(max=25.0) 以防止数值缩放作弊；引入 label_smoothing=0.1 缓解小 Batch Size 下的负样本匮乏问题，逼迫模型死磕物理表征。

3. 下游微调分类器重构 (DETR风格空间解耦)
槽位交叉注意力 (Slot Attention)：废弃会导致特征揉捏的“时序池化”，引入 DETR 风格的 6 个可学习查询向量 (Learnable Queries)。通过 Cross-Attention 机制，让 6 个吸盘从 150 帧的混合 WiFi 序列中主动剥离并解绑 6 个用户的独立特征，彻底解决多人动作特征的空间串扰（混叠）问题。

极速线性探测 (Fast Linear Probing)：在评估阶段彻底冻结 WiFi 预训练骨干 (requires_grad=False)，仅训练极轻量的槽位分类头。微调训练速度飙升 10 倍以上，且有效防止特征在 58% 空槽位的极端不平衡数据集中发生灾难性遗忘。

阈值门控 Argmax (Threshold-Gated Argmax)：修复纯 Argmax 导致“0用户样本”无中生有产生动作的物理漏洞。采用“先看最高概率是否突破 0.5，再取唯一最大值”的双重门控策略，完美兼顾“空房间全 0”与“防止多标签溢出”。

4. 数据管道与诊断工程
物理 ID 溯源与导出：打通 Dataloader，将真实的样本名（如 act_x_x）暴露给测试流程。

树状结果对比文件：在微调的前 5 轮，按“Sample -> 6个User槽位”的物理树状拓扑，将 Ground Truth 与预测动作矩阵无损导出至 txt 文件，极大提升底层 Debug 效率。

纯净日志打印：移除终端易导致乱码的 tqdm 进度条，替换为结构化、包含 Step 与 Batch Loss 的纯文本日志。
